# v0.2.2

**发布日期**: 2026-03-02

功能增强与架构优化版本。

## 新特性

### 意图识别与记忆增强

- **意图识别管道** - 实现分阶段意图识别机制：
  - 新增 `IntentPipeline` 模块（199 行）
  - 支持上下文重试机制，提升意图识别准确率
  - 集成到记忆检索流程，优化检索结果相关性
  - 移除旧版意图识别兼容层，统一使用新管道

- **记忆分类器** - 自动识别记忆类型：
  - 新增 `MemoryClassifier` 模块（343 行）
  - 自动分类：对话、工具调用、系统消息
  - 支持自定义分类规则

- **引用溯源系统** - RAG 级别引用溯源：
  - 新增 `CitationGenerator` 模块（165 行）
  - 自动生成知识库引用链接
  - 支持多格式引用展示（Markdown/纯文本）

### 知识库系统

- **知识库核心模块** - 完整的知识库管理系统：
  - 新增 `KnowledgeBase` 模块（1083 行）
  - 支持多格式文档解析：PDF、Word、Excel
  - 自动文档分块和向量化
  - 双层检索架构（向量 + 全文）

- **PDF 解析支持** - 新增 PDF 文档处理：
  - 集成 `pdf-parse@2` 依赖
  - 自动提取 PDF 文本内容
  - 智能分块存储

- **多嵌入模型支持** - 灵活的嵌入模型配置：
  - 新增 `VectorMigrationSystem` 模块（716 行）
  - 支持多嵌入模型并存
  - 自动向量迁移（无需手动重建）
  - 配置示例：`models.embed` 可配置多个模型

### 会话管理

- **SessionStore 集成** - 会话持久化支持：
  - 集成 SessionStore 实现会话历史持久化
  - 支持跨会话记忆保持
  - 自动恢复会话状态

### 日志可观测性

- **结构化日志系统** - 增强日志可观测性：
  - 新增 `LoggerConfig` 模块（713 行）
  - 结构化日志输出（JSON 格式）
  - 调用链追踪支持
  - 日志级别动态调整

- **日志追踪器** - 完整的调用链追踪：
  - 新增 `LoggerTracker` 模块（443 行）
  - LLM 调用追踪
  - 工具执行追踪
  - 记忆检索追踪

## 改进

- **记忆检索优化** - 集成意图识别管道，提升检索结果相关性
- **文档分块存储** - 优化文档分块策略，提高检索精度
- **会话历史持久化** - 从 JSONL 迁移到 SQLite，性能提升 10-100 倍
- **消息裁剪逻辑** - 优化消息裁剪策略，减少 token 消耗
- **CLI 状态显示** - 增强工具执行状态显示，提升用户体验

## 修复

- **空向量问题** - 修复空向量导致向量检索失败的问题
- **日志文件过大** - 日志按小时分割，解决单文件过大问题
- **tool 消息依赖** - 修复 tool 消息依赖问题
- **空会话检查** - 增强空会话场景处理，避免异常

## 重构

- **存储层迁移** - 知识库索引从 JSON 迁移到 SQLite：
  - 性能提升 10-100 倍
  - 支持并发读写
  - 更好的数据一致性

- **会话存储重构** - 会话存储从 JSONL 迁移到 SQLite：
  - 性能大幅提升
  - 支持复杂查询
  - 更好的扩展性

- **日志系统重构** - 提取命名常量并拆分超大函数：
  - 代码结构更清晰
  - 更易维护
  - 更易测试

- **工具安全防护** - 重构执行器，增强安全防护：
  - 路径访问控制增强
  - 文件系统工具安全限制
  - 更严格的参数校验

- **执行器重构** - 模块化设计，提升可维护性

## 用户影响

### 需要手动迁移

如果配置文件中使用了嵌入模型，首次启动会自动迁移向量数据：

```yaml
models:
  embed: openai/text-embedding-3-small
```

### 配置文件变化

新增多嵌入模型配置支持：

```yaml
models:
  embed:
    primary: openai/text-embedding-3-small
    secondary: ollama/nomic-embed-text
```
